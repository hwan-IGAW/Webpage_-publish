모바일인덱스(MI) 개편 방향 보고

― 시장 변화를 가장 먼저 읽는 ‘의사결정 인텔리전스’로 ―
2026 하반기 초도 보고  |  2026. 6.  |  작성: 모바일인덱스 전략
보고 요지
■MI를 ‘지표를 보는 서비스’에서 ‘의사결정을 돕는 인텔리전스’로 개편
■개편의 동력은 데이터의 확장·통합, 그 위에서 작동하는 변화 포착·해석 능력
○① 내부 데이터 통합 → ② 외부 데이터 결합 → ③ 시장 변화 포착 → ④ 보유 데이터 기반 해석
○네 단계를 거쳐 ‘의사결정 인텔리전스’로 포지셔닝
○향후 1st 파티 데이터 결합 시 진정한 의미의 의사결정 도구로 진화
■MI 3.0의 핵심 업무 = 3단계의 반복
○① 변화를 발견 → ② 원인·의미를 분석 → ③ 정기 보고·의사결정 자료로 자동화

Ⅰ.개편의 본질 ― 지표에서 의사결정으로

■현 위상 ― 정확하나 개별적으로 분리된 지표
○고객은 숫자는 확인하나, 숫자 간의 인과·맥락은 확인 불가
■개편 목표 ― 기업의 의사결정 도구 수준으로 격상
○‘무엇이 일어났는가’를 넘어 ‘왜 일어났고 무엇을 해야 하는가’까지 제공
■3대 전환 방향
○분석 도구(Why) ― 시장 심층 분석
○모니터링 도구(When) ― 시장 변화의 시의적 포착·해석
○실행 도구(Action) ― MI 직접 미수행, MI Ads(광고)·디파이너리(CRM) 연계
○MI 본체 = 분석 + 모니터링을 담당하는 의사결정 인텔리전스
■MI 3.0의 일하는 방식 ― 3단계 반복
○1단계 변화 발견 / 2단계 원인·의미 분석 / 3단계 정기 보고·의사결정 자료로 자동화
–이 3단계가 MI 3.0의 모든 기능을 관통하는 기준

Ⅱ.개편의 동력 ― 데이터의 확장과 통합

개편은 네 단계의 데이터 진화를 따라 진행 ― 각 단계가 다음 단계의 토대

■1단계내부 데이터 통합 ― 흩어진 자사 데이터를 하나로
○보유 자산 ― 5대 이종 인덱스(앱 사용량·카드 결제·앱마켓 매출·TV 시청·게임)
○현 문제 ― 각각 별도 상품으로 분리, 교차 분석이 번거로워 현실에서 미실현
○해법 ― MCP가 흩어진 데이터를 하나의 분석 체계로 만드는 기반
■MCP ― 통합의 방식
○모든 서비스를 하나의 거대한 데이터베이스로 다시 만드는 방식이 아님
○각 데이터 상품을 MCP로 연결, AI가 필요한 데이터를 호출·조합해 분석
○현재 모바일인덱스 MCP 출시, TV인덱스·소비인덱스·앱마켓 인덱스 순차 확대
○각 MCP 연결 시 고객은 서비스별 메뉴·데이터 구조를 이해할 필요 없음
○예) “경쟁사 A의 최근 성장이 광고 때문인지, 서비스 자체의 성장인지 분석” ― 한 질문으로 다음이 함께 활용
–앱 사용자·사용시간 변화 / TV 광고 집행량 / 앱마켓 매출 / 소비 데이터 / 앱 리뷰·주요 뉴스
○MCP의 가치 ― 개별 데이터를 하나의 ‘시장 분석 언어’로 연결
■2단계외부 데이터 결합 ― 관점의 확장
○공공·공시 데이터(주가·DART·통계청·국토부·기상청), SNS·커뮤니티·쇼핑몰 리뷰 결합
○무료 MCP/스킬셋 동반 공급 ― 자사 데이터와 외부 신호의 결합 용이화
○합성 소비자 인텔리전스(SCI, 알파 완료) 결합 시 분석 확장력 증대
■3단계시장 변화 포착 ― 상시 작동하는 레이더
○통합 데이터로 시장을 상시 감시
○급격한 변화 + 데일리·먼슬리로 미포착되는 완만한 구조 변화까지 포착
■4단계변화의 해석 ― 보유 데이터로 원인 설명
○포착에 그치지 않고 통합 데이터로 원인을 즉시 해석
○‘무엇 때문에, 어디서 변동했는가’를 제시 → 의사결정 인텔리전스의 본체
■진화1st 파티 데이터 결합
○애드브릭스·디파이너리 연계로 고객의 1st 파티 데이터(설치·행동·광고 성과) 결합
○시장 전체의 관점 위에 자사 고객의 실제 반응이 포개짐
○시장 해석을 넘어 ‘우리 회사는 무엇을 해야 하는가’에 답하는 의사결정 도구로 진화

Ⅲ.통합·결합으로 가능한 분석 (사례)

단일 지표로는 도출 불가, 데이터 통합·결합 시에만 가능한 분석

■내부 통합광고 효과의 전 과정 추적
○TV 광고 집행 →(TV인덱스) 도달 →(앱인덱스) 신규 설치·사용 증가 →(소비·앱마켓) 결제·매출 변화
○‘광고비의 매출 전환’ ― 기존에 답하기 어려웠던 질문에 대응
■내부 통합사회적 사건의 수요 영향 해석
○스타벅스 ‘탱크데이’ 불매 →(앱인덱스) 사용 급감 →(소비인덱스) 결제 위축 →(경쟁사) 반사이익을 연결
○월별 지표 변동의 원인을 데이터로 규명
■공공·공시 결합실적 발표 전 기업 체력의 선행 추정
○상장 유통사의 (앱인덱스) 사용자·(소비인덱스) 결제액 추세를 (DART 공시·주가)와 결합
○분기 실적 발표 이전 매출 방향을 선행 추정 → IR·투자·경쟁 대응의 근거 제공
■공공·공시 결합외부 변수 기반 수요 변동 설명
○(기상청) 기온·(통계청) 물가·소비심리를 (앱·소비 인덱스)와 결합
○예) 폭염·장마에 따른 배달·여행·패션 업종의 사용·결제 변동을 외부 충격으로 설명
■업종 분석시장 구조의 종합 해설
○여행 업종을 종합플랫폼·항공권·호텔/리조트로 분해, 소비·앱마켓·TV 결합 딥다이브
○자사뿐 아니라 소속 시장 전체의 구조를 제공
■소결
○상기 분석은 이론상 가능하나 데이터 분산으로 현실에서 미실현
○통합·결합 능력이 곧 차별점 ― 같은 데이터를 가져도 엮는 능력이 MI의 무기

Ⅳ.서비스 구현 ― MI 3.0 제품 구조

‘변화 발견 → 원인 분석 → 자동화’ 3단계를 따라 다음 구조로 구성

구분구성
①Today ― 로그인 첫 화면오늘의 중요 변화 / 자사·경쟁사·업종 요약전일·전주 대비 핵심 지표 / 추가 확인이 필요한 변화 / 최근 생성된 리포트
②Monitor ― 변화의 감시 (커스텀)관심 기업·앱·브랜드·업종의 변화 이력 등 커스텀 대시보드이상징후 및 알림 설정
③Analyze ― 원인의 분석이상징후·주요 변화에 대한 분석 / MCP 기반 비정형 대화형 분석분석 결과 저장 및 공유
④Reports ― 정기 보고의 자동화데일리 · 위클리 · 월간 · 분기 · 업종 연간 리포트분기·연간 리포트는 애널리스트 검수를 거치며, 업종 연간 리포트는 200만~500만원 백서 수준 리포트로 발간
⑤Data / Classic MI ― 기존 데이터 메뉴모바일인덱스 인사이트 · 소비인덱스 · 앱마켓 인덱스 · TV 인덱스 · 게임 인덱스
⑥Connect (추후) ― 실행으로의 연결디파이너리·애드브릭스 연계 / 외부 광고·CRM 솔루션
■전달·확장
○데일리·위클리·월간 요약을 음성/영상으로 Push
○모바일인덱스 모바일 앱 출시

Ⅴ.해자(Moat)

■데이터의 다종성·독점성
○앱 사용량·카드 결제·앱마켓 매출·TV 시청 데이터를 동시에 보유한 사업자는 국내에서 사실상 MI가 유일
○핵심이 되는 배타적 데이터는 공공·공개 데이터와 결합해도 그 독점성을 잃지 않음
■1st 파티 결합의 가능성
○어떤 유사 지표 서비스도 1st party 데이터를 자체 수급할 채널을 보유하지 못함 ― 그 역(逆) 또한 마찬가지
○디파이너리(DFN) MCP와 결합 시 강력한 경쟁력 확보 기대
○시장 데이터(공통) + 고객 데이터(개별)의 결합은 MI-DFN의 고유 영역
■SCI 데이터 결합 가능성
○특이사항·특이점을 드러낸 수치의 정량 해석을 넘어
○변화된 유저의 Signal과 Prediction이라는 정성적 영역까지 확장할 경우, 매우 강력한 해자로 작용 기대
■요약
○MI의 핵심 경쟁력은 기존에 결합이 어렵거나 불가능하던 데이터를 연결해 시장을 설명하는 강력한 능력

Ⅵ.기대 효과 ― 매출 전망

■기준선(현재) ― ARR 48억
○구독 ARR 45억 + 1회성 커스텀 데이터 3억 = 총 48억
○고객 120사 / 평균 구독 객단가 약 3,750만 / 연 이탈률 10~15%
○과거 3년 매출 연 15%↑의 동력은 1회성 커스텀 판매와 신규 라인 추가(게임·소비)였으며, 기존 라인 간 크로스셀은 미미 → 개편으로 풀려는 과제
■산출 전제 (기준 시나리오)
항목내용 / 근거
연 이탈률11% (보수 13% / 공격 9%) ― 구독 ARR에 적용
크로스셀분모 = 인사이트 단독 100사 / 누적 전환 1년 15%·3년 30% / 1사 증분 +2,400만(패키지 전환 6,000−3,600)
신규 순증1년 +40사 / 2년 +45 / 3년 +50 / 객단가 3,750만(현 평균 구독 객단가)
신규 달성 방식유입 증가가 아니라 단발→구독 전환↑·이탈↓ (월 3~5개사 신규 유입은 이미 지속)
1회성 커스텀연 3억, 연 15% 성장 (반복 매출 아님, 별도 인식)
크레딧(종량)1년 0 → 2년 구독의 5% → 3년 10%

○공격 시나리오 가정 시 3년차 ARR 약 122억
■성장 동력 우선순위
○① 단발→구독 전환·이탈 감소 (1년차 주력) ― 데이터 가치 상승으로 누수 차단
○② TV인덱스發 신규 클라이언트군 + 크로스셀 ― TV는 기존 디지털과 고객군이 달라 크로스셀 유효성 기대
○③ 크레딧 종량 매출 (2년차~) ― 사용량 기반 추가 과금
■가정의 한계 (검증 전제)
○순증 +40은 목표치(가설) ― 단발→구독 전환율·이탈률 개선 달성이 전제
○크레딧(종량) 매출은 모델 미확정 ― 2년차 이후 종량 과금 설계 필요

Ⅶ.시장 수요 근거

■지속적 신규 유입 ― 시장 수요는 이미 입증됨
○현재도 매월 3~5개사가 꾸준히 신규 유입 ― 10년 누적된 브랜드와 영업력 위에서 수요가 끊이지 않음
○다만 유입사 중 절반은 1개월 단발성, 1~2개사는 구독 이탈 → 순증이 0으로 보이던 구조
○즉 문제는 ‘수요 부족’이 아니라 ‘들어온 수요를 구독으로 묶지 못한 것’
○개편의 표적 = 단발→구독 전환 + 이탈 감소(누수 차단) → 같은 유입으로도 순증 전환 가능
–데이터 이용 가치가 오르면 단발 고객의 구독화·기존 고객의 잔존이 동시에 개선
○신규 고객 유치 기여도(예상) ― 서비스 강화에 따른 추가 유입 약 40%, 구독 전환·이탈 감소 효과 약 60%
■신규 세그먼트 신호 ― 대행사·투자사에서 수요 확인
○MI MCP 출시 직후 대행사·투자사가 가장 큰 관심을 표명 ― 다차원 데이터 결합의 최대 수요처로 부상 (= HSAS, TBWA, 그린브릭스, Altos 등)
○수요의 구조적 이유
–광고주: 자사·경쟁사 몇 곳만 추적(좁고 깊게)
–대행사·투자사: 시장 전체를 스캔(넓게) → 시장 분석·모니터링 니즈가 본질적으로 가장 높음
○개편 방향(시장 분석·모니터링 도구)과 최대 수요처가 정확히 일치 → 초기 타겟으로 설정
